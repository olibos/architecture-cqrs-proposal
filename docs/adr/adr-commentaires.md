# Privilégier le code auto-documenté

## Contexte
Chez Wavenet, nous avons l'habitude d'utilisation des outils de documentation automatique pour faciliter la compréhension de notre code par les développeurs et les équipes de maintenance. Cependant, nous avons constaté que cette approche n'était pas toujours efficace, car la documentation générée était parfois difficiles à comprendre ou n'étaient pas suffisamment précise pour expliquer les choix de conception et les décisions prises dans le code.

## Décision
Nous avons décidé de privilégier le code auto-documenté, c'est-à-dire le code qui se décrit de lui-même plutôt que de la documentation générée par un outil qui ne veut parfois rien dire. Nous encourageons nos développeurs à écrire du code clair et lisible, avec des noms de variables et de fonctions significatifs et des commentaires pertinents lorsque cela est nécessaire.

Nous avons également mis en place des revues de code régulières pour nous assurer que le code est compréhensible et maintenable par les autres membres de l'équipe.

## Conséquences
Nous sommes convaincus que cette approche nous permettra d'améliorer la qualité de notre code, de réduire le temps nécessaire à la maintenance et à la correction des bugs, et d'encourager la collaboration et la transparence au sein de l'équipe de développement. Cependant, nous sommes conscients que cette approche peut nécessiter plus d'efforts de la part des développeurs pour écrire du code clair et lisible, et peut également nécessiter des ajustements dans nos processus de développement.

Nous sommes prêts à investir ces efforts pour améliorer la qualité de notre code et espérons que cette approche contribuera à la satisfaction de nos clients et à la réussite de notre projet.
